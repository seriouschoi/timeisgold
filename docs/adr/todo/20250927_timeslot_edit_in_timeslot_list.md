## 기존 타임 슬롯 편집 기능.

드래그중인 정보를 intent에 담아서 뷰모델에 전달.
뷰 모델은 전달받은 값으로 상태를 갱신하여, 드래그를 표현함.

어..타임슬롯 드래그할때 policy일부를 써야해서..

지금보니.. policy에서 필요한 정보도 ui의 상태니깐..
uiState에 policy의 정책 값을 담아서 보내고..
